agenda(1) CHEATSHEET
---------------------
NAME
    agenda -- unified daily view: calendar.txt events + LIFE.TXT open items

SYNOPSIS
    agenda [-o|--only SECTION] [-c|--cal-days N] [-l|--agenda-lead N] [-h|--help]

SECTIONS PRINTED
    CALENDAR   events from calendar.txt, today .. today+CAL_DAYS
    TODO       open  :todo:    lines from LIFE.TXT
    WAITING    open  :waiting: lines from LIFE.TXT
    DUE        due:DATE items from LIFE.TXT, surfaced AGENDA_LEAD days
               early, loud once due or overdue
    (:someday: and :dropped: never appear here -- agenda only matches
     :todo: and :waiting:, by design)

OPTIONS
    -o, --only SECTION    print one section only
                             cal | calendar   calendar only
                             life             todo+waiting+due (no calendar)
                             todo             open todos only
                             waiting          open waiting-on's only
                             due              due: items only
                           default: all sections
    -c, --cal-days N      calendar horizon in days ahead     default 7
    -l, --agenda-lead N   due-soon lead time in days          default 2
    -h, --help            flag summary (built into the script itself)

    precedence: CLI flag  >  env var  >  built-in default

ENVIRONMENT
    LIFE_DIR     base dir                    default ~/life
    LIFE_FILE    LIFE.TXT path                default $LIFE_DIR/LIFE.TXT
    CAL_FILE     calendar.txt path            default $LIFE_DIR/stacks/calendar.txt
    CAL_DAYS     same as -c                   default 7
    AGENDA_LEAD  same as -l                   default 2

DUE MARKERS
    [DUE IN Nd]    due:DATE is inside the lead window, not due yet
    [DUE TODAY]    due:DATE == today
    [OVERDUE Nd]   due:DATE was Nd days ago
    Marker plus a leading ! render reverse-video on a tty; both fall
    back to plain ASCII automatically when piped (agenda | less,
    agenda > file) -- output is always strict ASCII, no box-drawing.

EXAMPLES
    agenda                  daily glance, everything
    agenda --only cal       what's on the calendar
    agenda --only life      LIFE.TXT items only (todo+waiting+due)
    agenda -o waiting       what am I blocked on
    agenda -c 30            monthly sweep of the calendar
    agenda -c 30 -l 5       wider horizon + earlier due-soon warning

GREP RECIPES
    For lookups agenda's own flags don't cover -- past the horizon, or
    narrowed by keyword instead of by section. 

    Tag-axis greps (state/area/project):
        grep ':todo:'      # finds all open actions; swap state as needed
        grep ':health:'    #finds all entries in that area
        grep -o ':project_[a-z0-9_]*:' | sort -u    # lists all

    Narrow agenda's live view by keyword:
        agenda | grep -i "dentist"
        agenda --only waiting | grep -i "refund"
        agenda --only life | grep ':project_taxes2026:'

    Calendar past CAL_DAYS -- agenda only shows a window, not the file:
        grep '^2026-08' ~/life/stacks/calendar.txt      # a given month
        grep -i "dentist" ~/life/stacks/calendar.txt    # any date, by keyword

    due: dates past AGENDA_LEAD -- same idea, direct in LIFE.TXT:
        grep 'due:'    # finds all dated actions; grep 'due:2026-' for a year
        grep 'due:2026-08' ~/life/LIFE.TXT               # a given month
        grep 'due:2026-08-1[0-5]' ~/life/LIFE.TXT        # a date range

FILES
    ~/life/LIFE.TXT               journal / notes / :tag: items -- see its
                                      own legend block for tag conventions
    ~/life/stacks/calendar.txt    dated events only, chronological, @tag

DEPLOYMENT
    machine-setup repo:  bin/agenda  -->  symlink  ~/.local/bin/agenda
    shell startup (.bashrc):  command -v agenda >/dev/null && agenda
